-- Prompt ---------------------------------------------------------------------------------
프로젝트의 전반적인 방향과 기능을 구체화하기 위해 주요 기능, 화면 구성, 기술 스택, 데이터

문서는 실제 개발에 바로 활용할 수 있는 수준으로 구성하고, 
결과는 마크다운 형식으로 docs/prd.md 파일로 작성해 주세요.

1. 주요 기능
    1) 이메일/비밀번호 로그인 및 회원가입: Supabase Auth를 활용해 사용자 인증 기능 제공
    2) 농장/구역 관리(CRUD)
        - 농장(온실·재배 구역)의 생성(Create), 조회(Read), 수정(Update), 삭제(Delete) 지원
        - 필드: 이름(name), 설명(description), 위치(location), 면적(area_sqm), 생성일(created_at), 
                수정일(updated_at), 소유 사용자(user_id)
    3) 센서 데이터 검색, 필터, 정렬 기능
        - 검색: 농장명·구역명, 센서명(또는 장비 ID)으로 검색 가능
        - 필터링: 센서 타입(온도/습도/조도/CO2/토양수분 등), 농장/구역, 기간(일자/시간 범위) 선택
        - 정렬: 수집일시순(최신/과거), 센서 타입순, 값 기준(높은순/낮은순)
    4) AI 환경 분석 및 권장 사항 기능: 사용자가 자연어로 질문하거나 버튼 클릭으로 AI가 농장/센서 데이터를 분석해 인사이트와 재배 개선 권장 사항 제공
        - 입력 예: "이번 주 1번 구역 온도 추이 어떻게 돼?"
        - 출력 예: 기간별 온도/습도 요약, 이상 구간 안내, 환기·가습·차광 등 재배 환경 개선 제안
    5) AI 요약 및 분석 기능: 버튼 클릭 한 번으로 AI가 전체 농장/센서 데이터를 분석해 요약 결과 제공
        - 일일 요약: 당일 구역별 센서 요약, 임계치 초과 이벤트 요약
        - 주간 요약: 지난 7일 환경 추이, 이상 구간 및 완료율·진행 상황 분석

2. 화면 구성
    1) 로그인/회원가입 화면: 사용자 인증 및 계정 관리
    2) 대시보드(메인) 화면: 농장/구역 목록, 실시간(또는 최신) 센서 요약, 검색·필터·정렬, AI 요약/질의 진입점
    3) 농장/구역 상세 화면: 해당 구역 CRUD, 연동 센서 목록, 기간별 센서 차트/테이블, AI 분석 호출
    4) 이후 확장: 통계 및 분석 화면(주간·월간 활동량, 구역별/센서별 통계, 완료율·이상 구간 시각화)

3. 사용 기술
    - Next.js 15(App Router), TypeScript(strict 권장), Tailwind CSS, Shadcn/ui, 
      Supabase(Auth/DB 등), AI SDK(Google Gemini API), ESLint(Next/TS 규칙 기반)

4. 데이터 구조(Supabase 활용)
    - users / profiles: 사용자 프로필 관리(Supabase Auth와 1:1 연동)
    - farms: 농장/구역 정보(사용자별 소유), users 테이블과 연결
    - sensors: 센서(장비) 메타정보, farms에 소속
    - sensor_readings: 센서별 시계열 측정값
    - alert_settings(확장): 임계치 및 알림 설정
    - alert_logs(확장): 알림 발생 이력

                create function public.handle_new_user()
                returns trigger as $$
                begin
                    insert into public.profiles (id, email, display_name)
                    values (new.id, new.email, new.email);
                    return new;
                end;
                $$ language plpgsql security definer;

                create trigger on_auth_user_created
                after insert on auth.users
                for each row
                execute procedure public.handle_new_user();

**1.2** Auth 설정: 이메일/비밀번호 사용 활성화
-------------------------------------------------------------------------------------------
    (1) Auth 이메일/비밀번호 로그인 켜기
        1) Supabase 프로젝트 대시보드 접속
        2) 왼쪽 메뉴에서 Authentication → Providers 클릭
        3) Email 섹션 선택
        4) 아래 옵션을 원하는 대로 설정:
            - Enable email provider: 체크 (ON)
            - Email auth:
                    Allow email signups (이메일 회원가입 허용) 체크
                    Enable email confirmations(이메일 확인 필수 여부)는
                        개발 초반에는 끄는 것도 편합니다.
                        운영 단계에서는 보안을 위해 켜는 것을 권장.
            - 비밀번호 관련 옵션은 기본값 유지해도 무방
            - 저장(Save) 버튼 누르면 됩니다.

    (2) 비밀번호 재설정(Password reset) 동작 확인
        1) 같은 Auth 메뉴에서 URL settings 또는 Redirect URLs 섹션 확인
        2) 비밀번호 재설정이나 이메일 확인 후 돌아올 사이트 주소를 등록
            예: http://localhost:3000/auth/callback (나중에 구현할 페이지 경로 기준으로 설정)
            (지금은 대략적인 개념만 잡고, 실제 경로는 나중에 로그인/회원가입 UI 만들 때 구체화해도 됩니다.)

    (3) 결과적으로 1.2가 “완료”된 상태란?
        1) Supabase Auth에서:
            - Email/Password Provider가 Enable 상태이고
            - 이메일 회원가입/로그인이 허용되어 있는 상태
       2)  이 상태가 되면, 프론트엔드에서:
            - await supabase.auth.signUp({ email, password })
            - await supabase.auth.signInWithPassword({ email, password })
